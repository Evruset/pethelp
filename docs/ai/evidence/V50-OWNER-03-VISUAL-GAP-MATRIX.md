## Black-band classification

Verdict: `CAPTURE_HARNESS_DEFECT`.

- Flutter runtime uses a non-transparent light root surface. The manifest-bound 375×812 Clinic PNG is opaque RGB; all 21,000 pixels in rows 0–55 were decoded and contain zero black pixels.
- The dark strip is repeated in prototype captures, including Catalog and Doctor controls, so it is not Clinic runtime state.
- The prototype `.skip-link` is fixed near the top and translated upward by only `120%`, leaving part of its dark rounded surface in captured pixels while unfocused.
- Direct hash navigation is captured without an explicit scroll reset or stable-frame condition.
- A cold diagnostic run reached the fixed 2200 ms capture deadline before `flutter-view` existed, proving first-frame/cache nondeterminism even though that attempt produced a blank light surface rather than a black band.
- The repair belongs to the evidence harness. Runtime cropping, fake status bars and black overlays are prohibited.

The capture regression must exercise both classifications: an exposed prototype skip link/direct-anchor offset is a capture defect; a transparent or black Flutter root surface is a runtime defect.
